Завдання 1 — Питання
Що таке cookie і навіщо вони потрібні?

Cookie — це маленький шматок тексту, який сервер відправляє браузеру, а браузер зберігає і відправляє назад при кожному запиті. Потрібні щоб сервер міг пам'ятати користувача — бо HTTP сам по собі не запам'ятовує нічого між запитами.
Яка різниця між Set-Cookie та Cookie?

Set-Cookie — це заголовок від сервера до браузера ("збережи це").

Cookie — це заголовок від браузера до сервера ("ось що ти мені колись дав").
Чи можуть cookies працювати без браузера?

Так. curl теж вміє — зберігає через -c файл і відправляє через -b файл. Просто робить це вручну, а не автоматично як браузер.

Завдання 2 — Питання
Чим відрізняються Session від Persistent Cookies?

Session — немає Expires/Max-Age, видаляється коли закриваєш браузер. Persistent — живе до конкретної дати або N секунд.
Що означає HttpOnly?

Забороняє JavaScript читати цю cookie. Захист від XSS — навіть якщо на сайті є шкідливий скрипт, він не зможе вкрасти сесію.
Чи може JS отримати HttpOnly cookie?

Ні, document.cookie її просто не покаже. В цьому і сенс атрибуту.
Різниця між Secure та HttpOnly?

Secure — cookie передається тільки по HTTPS (захист при передачі).

HttpOnly — cookie не читається через JS (захист на клієнті).

Це різні речі, часто ставлять обидва разом.

Завдання 3 — Питання
Який атрибут впливає на безпеку?

Одразу три: HttpOnly, Secure і SameSite — кожен захищає від свого типу атаки.
Якщо Path=/api, cookie відправиться на /api/users?

Так, відправиться. Браузер перевіряє чи шлях запиту починається з Path cookie, а /api/users починається з /api — все ок.
